* Previous experimentation involved **LSTM, GRU, and BERT** models.
* Comparative analysis determined **BERT is the best-performing model**.
* **MailFort will deploy only the BERT model**.
* **Explainability features (highlighted phrases, attention maps, etc.) are not implemented**.

This ensures **clean, robust, and academically defensible architecture**.

---

## SYSTEM GOAL

Build **MailFort**, a **Gmail-integrated intelligent email client** that:

1. Lets users sign in securely with **Google OAuth**.
2. Fetches emails from their Gmail inbox (server-side only).
3. Analyzes each email with **BERT** before displaying.
4. Shows **phishing/legitimate labels, confidence, and severity**.
5. Provides a **modern three-pane UI** with filters and indicators.

## ARCHITECTURE OVERVIEW

```
User
 ↓ Google OAuth
Next.js Server (BFF)
 ↓ Gmail API (read-only)
FastAPI (BERT Inference)
 ↓
Prediction Metadata
 ↓
MailFort UI (Three-Pane Email Client)
```

* Gmail API calls happen **only on the server**.
* OAuth tokens are **never exposed to the client**.
* BERT model is **loaded once at FastAPI startup**.

---

## CORE FUNCTIONAL PIPELINE

1. User logs in with Google OAuth.
2. Next.js server fetches inbox emails via Gmail API.
3. Emails are sent to **FastAPI** for analysis **before rendering**.
4. BERT model classifies emails as **phishing or legitimate**.
5. Prediction metadata attached:

   * `label`: phishing / legitimate
   * `confidence`: 0.0–1.0
   * `severity`: low / medium / high (derived from confidence)
   * `phishing_type`: optional / null
6. Emails rendered in **MailFort UI**.

---

## FASTAPI (BERT) ENDPOINT

**Endpoint:** `POST /analyze-email`

**Input**

```json
{
  "message_id": "string",
  "subject": "string",
  "from": "string",
  "body": "string"
}
```

**Output**

```json
{
  "label": "phishing | legitimate",
  "confidence": 0.0,
  "severity": "low | medium | high",
  "phishing_type": null
}
```

**Processing Pipeline**

* HTML stripping
* Signature removal
* Whitespace normalization
* Subject + body concatenation
* Token length truncation (BERT max length)
* Softmax probability extraction

**Severity Rule**

```text
confidence ≥ 0.90 → high
confidence ≥ 0.70 → medium
else → low
```
